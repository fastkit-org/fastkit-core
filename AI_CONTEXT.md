## Identity

FastKit Core is a lightweight toolkit that adds structure and common patterns to FastAPI
applications. It provides a modular repository pattern, service layer with lifecycle hooks,
Django-style query filters, event signals, HTTP utilities, caching, i18n, and validation
helpers — all designed to reduce infrastructure boilerplate and let developers focus on
features.

## Out of Scope

FastKit Core does NOT provide:
- Authentication or authorization (planned: `fastkit-auth`)
- Database migrations (use Alembic directly)
- Admin panel
- ORM (SQLAlchemy is a dependency, not an abstraction target)
- Background task queue (planned: `fastkit-worker`)
- GraphQL layer
- WebSocket management

## Architecture

### Pattern
Modular architecture. Each feature (users, invoices, etc.) lives in its own self-contained
module folder. There is no enforced Clean/Hexagonal split inside a module — pragmatism over
purity. Modules are flat and discoverable.

### Module Structure
Every module generated by `fastkit make module` follows this layout:

```
modules/
└── {entity_plural}/       # e.g. invoices/
    ├── __init__.py
    ├── models.py           # SQLAlchemy model(s)
    ├── schemas.py          # Pydantic request/response schemas
    ├── repository.py       # Data access — extends Repository or AsyncRepository
    ├── service.py          # Business logic — extends BaseCrudService or AsyncBaseCrudService
    ├── router.py           # FastAPI router — only HTTP concerns
    ├── signals.py          # Signal definitions (optional)
    └── listeners.py        # Signal receivers (optional)
```

NEVER merge layers into a single file (e.g. no "god service" with router logic inside).
NEVER put SQLAlchemy imports in `router.py`.
NEVER put `HTTPException` or `Request` imports in `service.py` or `repository.py`.

### Dependency Injection
Constructor injection ONLY.
NEVER: service locator, global registry, `ServiceClass()` inside another class body.

```python
# CORRECT
class InvoiceService(BaseCrudService[...]):
    def __init__(self, repository: InvoiceRepository) -> None:
        super().__init__(repository, response_schema=InvoiceResponse)

# WRONG — hidden dependency
class InvoiceService:
    def __init__(self):
        self.repository = InvoiceRepository(get_db())  # ← never do this
```

FastAPI `Depends()` is the wiring point. All DI happens in router `Depends()` chains, not
inside service or repository constructors.

## Naming Conventions

### Classes

| Type              | Convention                          | Example                        |
|-------------------|-------------------------------------|--------------------------------|
| SQLAlchemy model  | `{Entity}` (singular PascalCase)    | `Invoice`, `InvoiceItem`       |
| Pydantic create   | `{Entity}Create`                    | `InvoiceCreate`                |
| Pydantic update   | `{Entity}Update`                    | `InvoiceUpdate`                |
| Pydantic response | `{Entity}Response`                  | `InvoiceResponse`              |
| Repository        | `{Entity}Repository`                | `InvoiceRepository`            |
| Service           | `{Entity}Service`                   | `InvoiceService`               |
| Signal            | `{entity}_{event}` (snake_case)     | `invoice_created`              |

### Files

| File            | Contains                                   |
|-----------------|--------------------------------------------|
| `models.py`     | SQLAlchemy model(s) for this module        |
| `schemas.py`    | All Pydantic schemas for this module       |
| `repository.py` | Single repository class                    |
| `service.py`    | Single service class                       |
| `router.py`     | Single `APIRouter` instance + route funcs  |
| `signals.py`    | `Signal(...)` definitions                  |
| `listeners.py`  | Receiver functions connected to signals    |

### Methods (Repository / Service)

| Operation            | Method name                              |
|----------------------|------------------------------------------|
| Single by PK         | `get(id)` / `find(id)`                  |
| Single by PK or 404  | `get_or_404(id)` / `find_or_fail(id)`   |
| All records          | `get_all()`                              |
| Filtered list        | `filter(**kwargs)`                       |
| First match          | `first(**kwargs)` / `filter_one(**kwargs)` |
| Check existence      | `exists(**kwargs)`                       |
| Count                | `count(**kwargs)`                        |
| Page pagination      | `paginate(page, per_page)`              |
| Cursor pagination    | `cursor_paginate(cursor, per_page)`     |
| Create               | `create(data)`                           |
| Create many          | `create_many(data_list)`                 |
| Update               | `update(id, data)`                       |
| Update many          | `update_many(filters, data)`             |
| Delete               | `delete(id)` (soft if mixin present)     |
| Delete many          | `delete_many(filters)`                   |

## Core Components

### Base (SQLAlchemy declarative base)
**Module:** `fastkit_core.database.base`
**Class:** `Base`
**Purpose:** Declarative base for all models. Auto-generates `__tablename__` from class name
(PascalCase → snake_case plural). Provides `to_dict()`, `to_json()`, `update_from_dict()`.

USE: Every SQLAlchemy model must inherit from `Base`.
NEVER: Return `Base` subclass instances directly from routers — always map to Pydantic schema.

```python
from fastkit_core.database import Base
from sqlalchemy.orm import Mapped, mapped_column
from sqlalchemy import String

class Invoice(Base):
    # __tablename__ = "invoices" — auto-generated
    number: Mapped[str] = mapped_column(String(50), unique=True)
```

### Model Mixins
**Module:** `fastkit_core.database.mixins`
**Purpose:** Composable mixins to add common columns and behaviour.

| Mixin              | Adds                                                    |
|--------------------|---------------------------------------------------------|
| `IntIdMixin`       | `id: int` auto-increment primary key                   |
| `UUIDMixin`        | `id: UUID` primary key (auto-generated)                |
| `TimestampMixin`   | `created_at`, `updated_at` (auto-updated on save)      |
| `SoftDeleteMixin`  | `deleted_at`, `soft_delete()`, `restore()`             |
| `SlugMixin`        | `slug: str` unique indexed column                      |
| `PublishableMixin` | `published_at`, `publish()`, `unpublish()`, `schedule()`|

ALWAYS use `IntIdMixin` or `UUIDMixin` — `Base` does NOT include a default PK.
COMBINE mixins freely: `class Post(Base, IntIdMixin, TimestampMixin, SoftDeleteMixin)`.

```python
from fastkit_core.database import Base
from fastkit_core.database.mixins import IntIdMixin, TimestampMixin, SoftDeleteMixin

class Post(Base, IntIdMixin, TimestampMixin, SoftDeleteMixin):
    title: Mapped[str] = mapped_column(String(255))
```

### BaseWithTimestamps
**Module:** `fastkit_core.database.base_with_timestamps`
**Purpose:** Convenience base: `Base` + `IntIdMixin` + `TimestampMixin` pre-composed.
USE: When you want `id + created_at + updated_at` without listing mixins manually.

### Repository (sync)
**Module:** `fastkit_core.database.repository`
**Class:** `Repository[T]`
**Purpose:** Generic synchronous data access. Instantiate with `(model, session)`.

USE: Extend for entity-specific queries. Use directly for generic CRUD.
EXTEND: Add custom query methods. Call `self.session` for raw SQLAlchemy queries.
NEVER: Put business logic, validation, or HTTP concerns inside repository methods.
NEVER: Import or raise `FastKitException`/`HTTPException` in a repository.

```python
from fastkit_core.database import Repository
from sqlalchemy.orm import Session

class InvoiceRepository(Repository[Invoice]):
    def __init__(self, session: Session) -> None:
        super().__init__(Invoice, session)

    def find_by_number(self, number: str) -> Invoice | None:
        return self.first(number=number)
```

**Filter operators (Django-style):**
`field__eq`, `field__ne`, `field__lt`, `field__lte`, `field__gt`, `field__gte`,
`field__in`, `field__not_in`, `field__like`, `field__ilike`, `field__is_null`,
`field__between`, `field__startswith`, `field__endswith`, `field__contains`

```python
repo.filter(status='active', age__gte=18, _order_by='-created_at', _limit=20)
```

### AsyncRepository
**Module:** `fastkit_core.database.async_repository`
**Class:** `AsyncRepository[T]`
**Purpose:** Full async/await equivalent of `Repository`. Identical API, all methods are coroutines.

USE: In async FastAPI route handlers and async services.
NEVER: Mix with sync `Session` — requires `AsyncSession` from `sqlalchemy.ext.asyncio`.

```python
from fastkit_core.database import AsyncRepository
from sqlalchemy.ext.asyncio import AsyncSession

class InvoiceRepository(AsyncRepository[Invoice]):
    def __init__(self, session: AsyncSession) -> None:
        super().__init__(Invoice, session)
```

### BaseCrudService (sync)
**Module:** `fastkit_core.services.base_crud_service`
**Class:** `BaseCrudService[ModelType, CreateSchemaType, UpdateSchemaType, ResponseSchemaType]`
**Purpose:** Business logic layer with validation hooks, lifecycle hooks, and automatic
Pydantic response mapping.

USE: One service per module. Inject repository via constructor.
EXTEND: Override hook methods for custom logic — do NOT override CRUD methods directly.
NEVER: Import SQLAlchemy, `Session`, or ORM types directly in a service.
NEVER: Raise `HTTPException` in a service — raise `FastKitException` or `ValueError`.

**Hooks (override in subclass):**
- `validate_create(data)` — runs before create; raise to block
- `validate_update(id, data)` — runs before update; raise to block
- `before_create(data: dict) -> dict` — transform data before DB write
- `after_create(instance)` — post-creation side effects (events, emails)
- `before_update(id, data: dict) -> dict` — transform before update
- `after_update(instance)` — post-update side effects
- `before_delete(id)` — raise to block deletion
- `after_delete(id)` — cleanup after deletion

```python
from fastkit_core.services import BaseCrudService

class InvoiceService(BaseCrudService[Invoice, InvoiceCreate, InvoiceUpdate, InvoiceResponse]):
    def __init__(self, repository: InvoiceRepository) -> None:
        super().__init__(repository, response_schema=InvoiceResponse)

    def validate_create(self, data: InvoiceCreate) -> None:
        if self.exists(number=data.number):
            raise ValueError("Invoice number already exists")

    def after_create(self, instance: Invoice) -> None:
        invoice_created.send({"id": instance.id, "number": instance.number})
```

### AsyncBaseCrudService
**Module:** `fastkit_core.services.async_base_crud_service`
**Class:** `AsyncBaseCrudService[ModelType, CreateSchemaType, UpdateSchemaType, ResponseSchemaType]`
**Purpose:** Async equivalent of `BaseCrudService`. All hooks are coroutines.

USE: Pair with `AsyncRepository`. Hooks must be `async def`.

### Signal
**Module:** `fastkit_core.events.signal`
**Class:** `Signal`
**Purpose:** Lightweight in-process event system (pub/sub). Backend-swappable for broker
support in future versions.

USE: Define signals as module-level constants in `signals.py`. Send from service hooks.
NEVER: Send signals from routers or repositories.
NEVER: Put business logic inside a receiver — receivers orchestrate side effects only.

```python
# signals.py
from fastkit_core.events import Signal
invoice_created = Signal("invoice.created")
invoice_paid    = Signal("invoice.paid")

# listeners.py
from .signals import invoice_created

@invoice_created.connect
async def on_invoice_created(payload: dict, **kwargs) -> None:
    # send email, update stats, etc.
    ...

# Import listeners in main.py to register them:
import modules.invoices.listeners  # noqa
```

**Payload rule:** Always pass `dict`, `dataclass`, or Pydantic model. Plain objects
will trigger a `UserWarning` about broker incompatibility.

### Exceptions
**Module:** `fastkit_core.http.exceptions`
**Purpose:** Typed domain exceptions that map to HTTP status codes.

| Class                    | Status | When to use                        |
|--------------------------|--------|------------------------------------|
| `FastKitException`       | 400    | Base — extend for custom errors    |
| `NotFoundException`      | 404    | Resource not found                 |
| `ValidationException`    | 422    | Business validation failure        |
| `UnauthorizedException`  | 401    | Not authenticated                  |
| `ForbiddenException`     | 403    | Not authorized                     |
| `TooManyRequestsException`| 429   | Rate limit exceeded                |

RAISE in: service layer and below.
NEVER raise `HTTPException` (FastAPI) outside the `router.py` layer.

### Response Helpers
**Module:** `fastkit_core.http.responses`
**Purpose:** Standardized JSON response envelope.

```python
from fastkit_core.http.responses import success_response, error_response, paginated_response, cursor_paginated_response

# Single resource
return success_response(data=invoice, status_code=201)

# List with pagination metadata
items, meta = service.paginate(page=page, per_page=per_page)
return paginated_response(items=items, pagination=meta)

# Cursor pagination
items, next_cursor = service.cursor_paginate(cursor=cursor, per_page=per_page)
return cursor_paginated_response(items=items, next_cursor=next_cursor, per_page=per_page)

# Error
return error_response(message="Not found", status_code=404)
```

**Response envelope format:**
```json
{ "success": true, "data": {...} }
{ "success": true, "data": [...], "pagination": { "page": 1, "total": 100, ... } }
{ "success": false, "message": "...", "errors": {} }
```

## Anti-Patterns — NEVER Generate This

### Layer Violations

❌ FastAPI imports in service or repository layer
```python
# WRONG
from fastapi import HTTPException
class InvoiceService:
    def find_or_fail(self, id: int):
        raise HTTPException(status_code=404)  # ← never here

# CORRECT
from fastkit_core.http.exceptions import NotFoundException
class InvoiceService:
    def find_or_fail(self, id: int):
        raise NotFoundException("Invoice not found")
```

❌ SQLAlchemy model returned directly from router
```python
# WRONG
@router.get("/{id}")
def get_invoice(id: int, service: InvoiceService = Depends(...)):
    return service.find(id)  # returns Invoice model, not schema

# CORRECT — pass response_schema to service constructor
# service already maps to InvoiceResponse automatically
```

❌ Business logic inside router function
```python
# WRONG
@router.post("/")
def create_invoice(data: InvoiceCreate, session: Session = Depends(get_db)):
    if session.query(Invoice).filter_by(number=data.number).first():
        raise HTTPException(422, "Duplicate number")
    ...

# CORRECT — put this in service.validate_create()
```

❌ Direct repository calls from router
```python
# WRONG
@router.get("/")
def list_invoices(repo: InvoiceRepository = Depends(...)):
    return repo.get_all()

# CORRECT — always go through service
```

### Dependency Issues

❌ Instantiating service inside another service
```python
# WRONG
class OrderService:
    def __init__(self, repo):
        self.invoice_service = InvoiceService(...)  # hidden coupling

# CORRECT — inject via constructor
class OrderService:
    def __init__(self, repo, invoice_service: InvoiceService) -> None:
        self.invoice_service = invoice_service
```

❌ Using global session or global service instance
```python
# WRONG
db = SessionLocal()
user_service = UserService(Repository(User, db))

# CORRECT — use FastAPI Depends() with scoped session
```

### Error Handling

❌ Swallowing exceptions silently
```python
# WRONG
try:
    service.create(data)
except Exception:
    pass

# CORRECT
try:
    service.create(data)
except ValueError as e:
    return error_response(str(e), status_code=422)
```

❌ Returning `None` from service without documenting it
```python
# WRONG — caller doesn't know if None means "not found" or "error"
def find(self, id): ...

# CORRECT — use find_or_fail() when None is not acceptable
```

### Data Flow

❌ Accepting ORM model as input to service
```python
# WRONG
def update(self, instance: Invoice, data: dict): ...

# CORRECT
def update(self, id: int, data: InvoiceUpdate): ...
```

❌ Skipping response_schema — returning raw ORM instances from service
```python
# WRONG
class InvoiceService(BaseCrudService[...]):
    def __init__(self, repo):
        super().__init__(repo)  # response_schema=None → returns ORM model

# CORRECT (unless intentional and documented)
    def __init__(self, repo):
        super().__init__(repo, response_schema=InvoiceResponse)
```

❌ Soft-deleted records appearing in results
```python
# WRONG — manual query without soft-delete filter
stmt = select(Invoice)  # includes deleted records

# CORRECT — Repository/AsyncRepository filters deleted_at automatically
# when SoftDeleteMixin is present on the model
```

### Signals

❌ Connecting receivers without importing listeners
```python
# WRONG — signals.py defines Signal, but listeners.py is never imported
# receivers are never registered, signals fire into the void

# CORRECT — in main.py or app startup:
import modules.invoices.listeners  # noqa
```

❌ Sending non-serializable payload
```python
# WRONG
await invoice_created.send(invoice_instance)  # SQLAlchemy object

# CORRECT
await invoice_created.send({"id": invoice.id, "number": invoice.number})
```

## How to Add a New Feature

When asked to create a new module (e.g. "create Product module"):

**Step 1 — Scaffold with CLI (preferred)**
```bash
fastkit make module Product           # sync
fastkit make module Product --async   # async
fastkit make module Product --async --signals  # async + events
```

**Step 2 — If scaffolding manually, follow this order:**
1. `modules/products/models.py` — SQLAlchemy model, choose mixins
2. `modules/products/schemas.py` — `ProductCreate`, `ProductUpdate`, `ProductResponse`
3. `modules/products/repository.py` — extend `Repository` or `AsyncRepository`
4. `modules/products/service.py` — extend `BaseCrudService` or `AsyncBaseCrudService`
5. `modules/products/router.py` — `APIRouter`, inject service via `Depends()`
6. `modules/products/signals.py` + `listeners.py` — if events needed
7. Register router in `main.py`
8. Import listeners in `main.py` if signals present
9. Create Alembic migration: `fastkit migrate make -m "create_products"`

NEVER skip steps or merge layers.
NEVER create a module without at least `models.py`, `schemas.py`, `repository.py`, `service.py`, `router.py`.

## CLI Reference (fastkit-cli)

### `fastkit make module <Name>`
Generate complete module scaffold.

| Flag          | Description                                 |
|---------------|---------------------------------------------|

Generate only `models.py`.

### `fastkit make schema <Name>`
Generate only `schemas.py`.

### `fastkit make repository <Name> [--async]`
Generate only `repository.py`.

### `fastkit make service <Name> [--async]`
Generate only `service.py`.

### `fastkit make router <Name> [--async]`
Generate only `router.py`.

### `fastkit make signals <Name>`

### `fastkit migrate make -m "<message>"`
Create Alembic migration.

### `fastkit migrate up`
Apply pending migrations.

### `fastkit server`
Start development server (uvicorn with reload).

## Testing Conventions

### Structure
```
tests/
├── unit/           # service + repository logic, no real DB, no HTTP
├── integration/    # real DB (SQLite in-memory or test DB), no HTTP
└── e2e/            # full stack via FastAPI TestClient
```

### Rules
- Unit tests NEVER hit a real database — mock repository with `pytest-mock` or `MagicMock`
- Integration tests use `aiosqlite` or SQLite in-memory via SQLAlchemy
- E2E tests use FastAPI `TestClient` or `AsyncClient` (httpx)
- Name test files: `test_{module_name}.py`
- Name test classes: `Test{ClassName}`
- Use `pytest` fixtures for session, service, and repository setup
- Async tests use `pytest-asyncio` with `asyncio_mode = "auto"`

### Mocking pattern for service unit tests
```python
from unittest.mock import MagicMock
from modules.invoices.service import InvoiceService
from modules.invoices.schemas import InvoiceCreate

def test_create_raises_on_duplicate(mock_repo):
    mock_repo.exists.return_value = True
    service = InvoiceService(mock_repo)

    with pytest.raises(ValueError, match="already exists"):
        service.create(InvoiceCreate(number="INV-001", ...))
```

### `modules/invoices/router.py` — dependency wiring
```python
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession
from fastkit_core.database.session import get_async_db
from fastkit_core.http.responses import success_response, paginated_response
from .service import InvoiceService
from .schemas import InvoiceCreate, InvoiceUpdate

router = APIRouter()

def get_service(db: AsyncSession = Depends(get_async_db)) -> InvoiceService:
    return InvoiceService(InvoiceService(db))

@router.get('/')
async def list_invoices(page: int = 1, service: InvoiceService = Depends(get_service)):
    items, meta = await service.paginate(page=page, per_page=20)
    return paginated_response(items=items, pagination=meta)

@router.post('/', status_code=201)
async def create_invoice(data: InvoiceCreate, service: InvoiceService = Depends(get_service)):
    invoice = await service.create(data)
    return success_response(data=invoice, status_code=201)
```

### `alembic/env.py` — required snippet for migrations
```python
import os
import sys
from logging.config import fileConfig

from alembic import context
from sqlalchemy import engine_from_config, pool

# Make project root importable
sys.path.insert(0, os.path.realpath(os.path.join(os.path.dirname(__file__), '..')))

from fastkit_core.config import ConfigManager
from fastkit_core.database import Base, build_database_url

# Alembic Config object — do NOT shadow this with your own 'config' variable
alembic_config = context.config
if alembic_config.config_file_name is not None:
    fileConfig(alembic_config.config_file_name)

# Import ALL models here so Alembic autogenerate detects them
from modules.invoices.models import Invoice  # noqa
# from modules.products.models import Product  # noqa

target_metadata = Base.metadata


def run_migrations_online() -> None:
    configuration = ConfigManager(modules=['database'])
    url = build_database_url(configuration)  # always sync URL for Alembic

    connectable = engine_from_config(
        {'sqlalchemy.url': url},
        prefix='sqlalchemy.',
        poolclass=pool.NullPool,
    )
    with connectable.connect() as connection:
        context.configure(
            connection=connection,
            target_metadata=target_metadata,
            compare_type=True,
        )
        with context.begin_transaction():
            context.run_migrations()


if context.is_offline_mode():
    pass  # implement run_migrations_offline() if needed
else:
    run_migrations_online()
```

### ConfigManager key rules
- `modules` list maps to files in the `config/` package: `['app', 'database']` loads `config/app.py` and `config/database.py`
- Access values with dot notation: `config.get('database.CONNECTIONS')`, `config.get('app.DEBUG')`
- `.env` is auto-discovered from CWD upward — no explicit path needed unless using non-standard location
- Environment variables override config files: `DATABASE_CONNECTIONS` overrides `database.CONNECTIONS`
- NEVER instantiate `ConfigManager` inside a repository, service, or router — instantiate once at app startup and pass via DI

### Database session rules
- Call `init_database(config)` or `init_async_database(config)` ONCE at startup, before the app starts serving requests
- Use `get_db` / `get_async_db` as FastAPI `Depends()` in routers to get a scoped session per request
- Use `get_read_db` / `get_async_read_db` for read-heavy endpoints when read replicas are configured
- Call `shutdown_database()` / `shutdown_async_database()` in the `lifespan` context manager on teardown
- NEVER call `init_database` inside a request handler, service, or repository

from pathlib import Path
